## Rule
NeonAvatar.tsx uses a static `Record<number, any>` require() map for 15 individual PNG files.
Type must be `any`, NOT `ReturnType<typeof require>` — the latter causes TS2769.

## Avatar files — named PNGs (512×512 each)
Each file is a clean individual asset generated by `scripts/generate-avatars.mjs` using sharp + SVG.
NO sprite-sheet cropping. NO background-position. Always use `resizeMode="contain"`.

## Regeneration
To regenerate all 15 PNGs (e.g. after style changes):
```
node scripts/generate-avatars.mjs
```
Each named .png is also written to avatar_N.png for legacy compat.

**Why:** Sprite-sheet crops were incorrect (bad coordinates), causing half-icons and neighbor bleed.
Named individual PNGs guarantee exactly one symbol per file with no cropping.
**How to apply:** Any new avatar PNG must be added to AVATAR_IMAGES with a literal require().
Metro bundler requires static literal require() calls — no computed paths.
