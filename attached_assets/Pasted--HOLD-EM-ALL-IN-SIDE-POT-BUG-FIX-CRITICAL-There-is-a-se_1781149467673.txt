# HOLD'EM ALL-IN / SIDE POT BUG FIX (CRITICAL)

There is a serious payout calculation issue in Texas Hold'em when players go all-in with different stack sizes.

The current system incorrectly awards chips as if both players were able to fully match each other's wagers.

This is NOT how poker works.

The game must use proper side-pot and all-in logic.

--------------------------------------------------

EXAMPLE OF THE BUG

Player A:
500,000 chips

Player B:
138,000 chips

Player A goes ALL-IN.

Player B calls ALL-IN for 138,000.

Current behavior:

The winner is being awarded as if both players contributed 500,000.

This is incorrect.

--------------------------------------------------

CORRECT BEHAVIOR

Player B can only risk:

138,000

Player A can only win:

138,000 from Player B.

The remaining:

362,000

must immediately be returned to Player A before showdown.

The unmatched portion is never placed into the pot.

The larger stack cannot force the smaller stack to play for chips they do not possess.

This follows standard poker all-in and side-pot rules. A player can only win chips that were actually matched by opponents. (Wikipedia)

276,000
