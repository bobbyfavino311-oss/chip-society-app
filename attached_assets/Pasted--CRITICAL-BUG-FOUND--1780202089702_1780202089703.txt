===========================================================
CRITICAL BUG FOUND
===========================================================

The avatar issue is NOT a centering issue.

The issue is that the app is still cropping from a sprite sheet.

Evidence:

Some avatars are perfectly centered:

- Martini Glass
- Neon Palm
- Champagne
- Shark Fin

Others are offset:

- Dice Stack
- Cassette
- Lightning Bolt
- Flamingo
- Poker Chip
- Tiger Eye

If centering were broken, ALL avatars would be offset equally.

Instead, every avatar is offset differently.

This proves the system is still using crop coordinates from a larger reference sheet.

===========================================================
STOP USING SPRITE SHEET CROPPING
===========================================================

Do not display avatars by cropping sections from a master image.

Do not use x/y crop coordinates.

Do not use background-position offsets.

Do not use sprite rendering.
