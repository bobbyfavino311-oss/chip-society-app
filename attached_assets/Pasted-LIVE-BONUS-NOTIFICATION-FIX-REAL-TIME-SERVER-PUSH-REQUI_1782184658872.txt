LIVE BONUS NOTIFICATION FIX — REAL-TIME SERVER PUSH REQUIRED

The admin bonus system is still not sending notifications live.

Current issue:

Admin gives player chips from the admin page, but the player does not receive the notification in real time.

This must be fixed.

REQUIRED BEHAVIOR

When admin gives a player a casino bonus:

1. Server adds chips to player balance.
2. Server creates transaction log.
3. Server creates notification record.
4. If player is online, server pushes notification instantly.
5. Player receives popup immediately without refreshing.
6. Player balance updates immediately.
7. Notification center badge updates immediately.

IMPORTANT

Do NOT rely only on database polling.

Do NOT require player to refresh.

Do NOT require logout/login.

Do NOT wait until app restart.

This must be real-time.

Use one of:

• WebSocket
• Socket.IO
• Server-Sent Events
• Realtime database listener
• Push notification system
